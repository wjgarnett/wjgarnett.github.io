1. 简介

日常学习和工作中养成记笔记的好习惯对自我成长帮助巨大。用自己的语言去阐释学习的知识并输出是检验我们对知识点理解是否透彻的最佳方法。把我们记录的知识点通过个人文档网站的方式分享给读者既可以确保笔记质量又能收到反馈，加深自己对知识点的理解。

个人采用markdown记笔记，并借助MkDocs+github分享发布自己的笔记。这里主要记录在已有markdown笔记的情况下，用MkDocs+github搭建文档网站的方法。

2. 环境搭建

2.1 环境安装

Mkdocs是一个python库，可将markdown文本生成html网页。

pip install mkdocs
pip install mkdocs-material
pip install mkdocs-material-extensions

2.2 基本使用

mkdocs new wjgarnett.github.io
cd ./wjgarnett.github.io
mkdocs serve #开启网页服务，主要用来进行本地实时调试
mkdocs build #构建html网页
mkdocs --help

git init
git add .
touch README.md .gitignore #对应文件中添加相应内容
touch .gitignore #忽略site/目录
...
git pull -u origin master

mkdocs gh-deploy #发布到internet，可通过2.3的自动化方式替代

2.3 github自动化构建网页设置

mkdocs gh-deploy命令式发布的方式比较繁琐，我们可以在对应github的仓库的action选项里设置自动化构建工作流。

# This is a basic workflow to help you get started with Actions

name: public site

# Controls when the workflow will run
on:
  # Triggers the workflow on push or pull request events but only for the main branch
  push:
    branches: [ master ]
  pull_request:
    branches: [ master ]

  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

# A workflow run is made up of one or more jobs that can run sequentially or in parallel
jobs:
  # This workflow contains a single job called "build"
  build:
    # The type of runner that the job will run on
    runs-on: ubuntu-latest

    # Steps represent a sequence of tasks that will be executed as part of the job
    steps:
      # Checks-out your repository under $GITHUB_WORKSPACE, so your job can access it
      - uses: actions/checkout@v3

      - name: Install mkdocs
        run: |
          python -m pip install mkdocs==1.2.2
          python -m pip install mkdocs-material==7.1.11
          python -m pip install mkdocs-material-extensions==1.0.1
          python -m pip install mike==1.0.1

      - name: Create html
        run: mkdocs build

      - name: Public
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./site
